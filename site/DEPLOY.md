# Deploying The Landing Page

The site is a single static HTML file. Do not deploy it until the package source,
public links, and evidence bundle story are coherent.

## Cloudflare Pages

1. Push the canonical repo.
2. In Cloudflare Pages, create a project from Git.
3. Set the build output directory to `site`.
4. Add the custom domain in Pages settings.

## Required Edits Before Going Live

- [ ] Replace the contact email if needed.
- [ ] Add a public GitHub link once the canonical repo exists.
- [ ] Add a paper or benchmark link only if it is actually public.
- [ ] Confirm the page still says hosted team features are future work.
- [ ] Confirm the page frames the benchmark as synthetic evidence.
